📚 Bagaimana 4 Pilar Keamanan Bekerja di {{ app_title }}

1. Keamanan (Kerahasiaan) — Setiap pengiriman file membuat pasangan kunci ECC ephemeral (sekali pakai). Pengirim & penerima sama-sama menjalankan ECDH dengan kunci masing-masing sehingga mendapat shared secret yang identik tanpa pernah mengirimnya lewat jaringan. Shared secret itu diregangkan dengan HKDF-SHA256 menjadi kunci AES-256, lalu file dienkripsi dengan AES-256-GCM.

2. Integritas — AES-GCM punya "auth tag" bawaan: kalau ciphertext diubah walau 1 bit, dekripsi akan gagal total. Sebagai lapis kedua, hash SHA-256 dari file asli juga dikirim terpisah dan dicocokkan ulang setelah dekripsi.

3. Autentikasi — Pengirim menandatangani hash file dengan kunci PRIVAT identitasnya (ECDSA). Penerima memverifikasi tanda tangan itu memakai kunci PUBLIK pengirim yang tercatat di direktori server relay saat registrasi.

4. Non-Repudiasi — Karena tanda tangan ECDSA hanya bisa dibuat oleh pemegang kunci privat, dan kunci publik yang dipakai untuk verifikasi dicocokkan dengan direktori server (bukan sekadar dari paket itu sendiri), pengirim tidak bisa menyangkal telah mengirim file tersebut — kecuali kunci privatnya bocor.

Catatan: server relay (mode HOST) hanya meneruskan paket terenkripsi dan menyimpan daftar kunci publik. Server tidak pernah melihat isi file asli maupun kunci AES.
